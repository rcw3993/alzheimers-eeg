# EEG Alzheimer's Detection

Machine learning pipeline for early Alzheimer's disease detection using EEG signals.
Compares multiple signal representations (bandpower, STFT, PLV connectivity, raw)
paired with appropriate model architectures, using subject-wise cross-validation
to prevent data leakage.

## Results summary

| Representation | Model | CV AUC | Electrodes |
|---|---|---|---|
| Bandpower | Random Forest | 0.815 В± 0.04 | 5 (top-k sweep) |
| Connectivity (PLV) | Random Forest | 0.769 В± 0.05 | 19 |
| STFT | 2D CNN | 0.747 В± 0.06 | 19 |
| Raw windows | 1D CNN | 0.508 В± 0.07 | 19 |

Final model (5-electrode bandpower RF, threshold-optimized): **91% sensitivity, ~72% specificity**.

## Usage

### Option A: Full experiment in one command

```bash
python scripts/run_experiment.py configs/experiments/bandpower_rf_v1.yaml
```

This runs feature extraction then training end-to-end.
If features are already extracted, skip that step:

```bash
python scripts/run_experiment.py configs/experiments/bandpower_rf_v1.yaml --skip-extraction
```

### Option B: Step by step

**Step 1 вЂ” Extract features for a representation:**
```bash
python scripts/extract_features.py configs/features/bandpower.yaml
python scripts/extract_features.py configs/features/stft.yaml
python scripts/extract_features.py configs/features/connectivity.yaml
python scripts/extract_features.py configs/features/raw.yaml

# Override subjects on the command line (e.g. test on 3 subjects first)
python scripts/extract_features.py configs/features/bandpower.yaml --subjects 1 2 3
```

**Step 2 вЂ” Train and evaluate:**
```bash
python scripts/train.py configs/training/bandpower_rf.yaml
python scripts/train.py configs/training/bandpower_rf_thresh.yaml
python scripts/train.py configs/training/stft_cnn.yaml
python scripts/train.py configs/training/connectivity.yaml

# Also save a final model trained on all data
python scripts/train.py configs/training/bandpower_rf_thresh.yaml --save-model
```

## Adding a new representation

1. Add a `compute_X(windows_data, sfreq, **kwargs)` function in `src/features/X.py`
2. Register it in `scripts/extract_features.py`'s `REPRESENTATION_FN` dict
3. Subclass `BaseEEGDataset` in `src/data/datasets.py` (implement `_pt_suffix` and `_load_window`)
4. Add a `configs/features/X.yaml`

## Adding a new dataset

1. Drop the dataset folder under `data/` (e.g. `data/ds002722/`)
2. Update `src/data/loaders.py` вЂ” add a `load_raw_eeg_ds002722()` or generalize `load_raw_eeg()` to accept a `dataset` argument
3. Update `get_diagnosis()` or add a dataset-specific equivalent
4. Create new feature configs pointing at the new subjects

## Reproducibility

All experiments are seeded (`set_seed(42)` in `src/utils.py`).
Subject-wise `GroupKFold` ensures no subject appears in both train and test splits.
Configs are the single source of truth for all hyperparameters вЂ” no hardcoded values in source files.
